## 2024-05-24 - Avoid runBlocking in ExoPlayer main thread\n**Learning:** The application uses ExoPlayer for media playback. Using runBlocking to synchronously read DataStore within ExoPlayer main-thread callbacks (e.g. onMediaItemTransition, onPlaybackStateChanged) causes UI stutters and potential ANRs. This happens because these callbacks are fired on the main thread, and synchronously blocking it to read from disk (DataStore) halts the UI.\n**Action:** Always read DataStore asynchronously in a coroutine scope (e.g., using a Flow and caching the value, or reading it asynchronously before it's needed) and update local variables that the ExoPlayer callbacks can access instantly.
