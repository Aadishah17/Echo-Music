## 2024-05-18 - Replacing `runBlocking` in ExoPlayer Main-Thread Callbacks
**Learning:** Using `runBlocking` inside `MusicService.kt` to read `dataStore` during ExoPlayer callbacks (`onMediaItemTransition`, `onPlaybackStateChanged`) blocks the main thread, causing ANRs and stuttering UI. The `DataStore` reading operations, especially `runBlocking(Dispatchers.IO) { data.first()[key] }`, are synchronous and heavy. ExoPlayer's property (e.g. `player.repeatMode`, `player.shuffleModeEnabled`) are kept perfectly in sync with the `DataStore` preferences so it's a 1-to-1 mapping.
**Action:** Replace `runBlocking { dataStore.get(RepeatModeKey, REPEAT_MODE_OFF) }` with `player.repeatMode` and `runBlocking { dataStore.get(ShuffleModeKey, false) }` with `player.shuffleModeEnabled` within `MusicService.kt` callbacks. This avoids main-thread blocking and provides instant access to the current state.
