## 2024-05-18 - Compose Performance Optimization\n**Learning:** When collecting state flows in Jetpack Compose, using `collectAsState` without lifecycle awareness means flows will continue emitting updates even when the app is in the background, consuming CPU and battery. \n**Action:** Use `collectAsStateWithLifecycle()` from `androidx.lifecycle.compose` to ensure flow collection stops when the screen is hidden, making backgrounding the app much more efficient.
## $(date +%Y-%m-%d) - ExoPlayer Main Thread DataStore Reads
**Learning:** Using `runBlocking` to read Jetpack DataStore preferences (like repeat/shuffle state) synchronously during ExoPlayer main-thread callbacks (`onMediaItemTransition`, `onPlaybackStateChanged`) blocks the main thread, causing UI stutters, audio glitches, and ANRs during track transitions.
**Action:** Always read `player.repeatMode` and `player.shuffleModeEnabled` instead of hitting DataStore, as they are kept in sync by ExoPlayer. For other preferences, wrap reads in `scope.launch(Dispatchers.IO)` and ensure ExoPlayer accesses (like `player.getMediaItemAt`) are done on the main thread before context switching to IO.
