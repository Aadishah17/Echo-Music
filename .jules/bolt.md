## 2024-06-16 - Compose `items` parameter optimization
**Learning:** In Compose Lazy lists (`LazyColumn`, `LazyRow`, `LazyVerticalGrid`), performing data transformations (like `.distinctBy { it.id }`, `.filterIsInstance<T>()`, `.map { ... }`, `.shuffled()`) directly within the `items(...)` block causes those expensive operations to be re-run on *every single recomposition* of the parent list component, rather than just when the underlying data changes.
**Action:** Always pre-compute these transformed lists outside the Lazy list block, preferably using `remember` with appropriate keys to memoize the result, and pass the already-transformed list into `items(items = myComputedList)`. This avoids redundant computation and unnecessary memory allocation during scroll/recomposition.
