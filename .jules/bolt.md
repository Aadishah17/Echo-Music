## 2024-05-18 - Compose Performance Optimization\n**Learning:** When collecting state flows in Jetpack Compose, using `collectAsState` without lifecycle awareness means flows will continue emitting updates even when the app is in the background, consuming CPU and battery. \n**Action:** Use `collectAsStateWithLifecycle()` from `androidx.lifecycle.compose` to ensure flow collection stops when the screen is hidden, making backgrounding the app much more efficient.

## 2024-05-18 - ExoPlayer Main Thread Performance
**Learning:** Using `runBlocking` to synchronously read from DataStore within ExoPlayer's main-thread callbacks (e.g. `onMediaItemTransition`, `onPlaybackStateChanged`) blocks the main UI thread. This causes severe UI stutters and ANRs (Application Not Responding) during playback transitions. ExoPlayer provides cached properties like `player.repeatMode` and `player.shuffleModeEnabled` which should be used instead of reading from DataStore synchronously.
**Action:** Replaced `runBlocking` reads of DataStore in ExoPlayer listeners with direct reads of `player.repeatMode` and `player.shuffleModeEnabled`. For other preferences, wrapped them in `scope.launch(Dispatchers.IO)` to run asynchronously.
