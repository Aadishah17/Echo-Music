## 2024-05-18 - Compose Performance Optimization\n**Learning:** When collecting state flows in Jetpack Compose, using `collectAsState` without lifecycle awareness means flows will continue emitting updates even when the app is in the background, consuming CPU and battery. \n**Action:** Use `collectAsStateWithLifecycle()` from `androidx.lifecycle.compose` to ensure flow collection stops when the screen is hidden, making backgrounding the app much more efficient.
\n## 2026-07-09 - ExoPlayer Main-Thread DataStore Blocking\n**Learning:** In ExoPlayer listeners like `onMediaItemTransition` and `onPlaybackStateChanged`, reading Jetpack DataStore synchronously via `runBlocking` blocks the main thread, causing UI stutters and potential ANRs. This happens because these callbacks are fired heavily on the main thread during playback events.\n**Action:** Instead of `runBlocking { dataStore.get(Key) }` for player properties, read directly from ExoPlayer's cached properties (e.g., `player.repeatMode`, `player.shuffleModeEnabled`) which are kept in sync with DataStore.
