## 2024-05-24 - Avoid Iterator Allocation with fastAny in Compose
**Learning:** Using standard collection functions like `.any { ... }` or `.forEach { ... }` on Kotlin `List` types allocates a new `Iterator` object under the hood for each invocation. When filtering large lists on the UI thread or inside `remember { ... }` blocks (which run frequently or synchronously during compositions/recompositions), these allocations add up and cause unnecessary memory churn and Garbage Collection pressure, slightly degrading UI performance. Compose UI Util provides `.fastAny`, `.fastForEach`, etc., which are inline functions that use index-based `while` loops instead of iterators, completely avoiding the allocation.
**Action:** Always prefer `androidx.compose.ui.util.fastAny`, `fastForEach`, `fastMap`, etc. over standard library equivalents when iterating over lists in UI code (especially inside UI state calculations like `remember` blocks filtering lists).
